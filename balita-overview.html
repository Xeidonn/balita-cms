What is this?

A writing and publishing tool for bilingual newsrooms

Balita CMS is a content management system built specifically for small editorial teams that publish in both English and Filipino. It handles the full process — from writing to translation to publishing — in one place.

Think of it as a newsroom dashboard: writers submit stories, an editor checks the Filipino translation, and with one click the article goes live in both languages.

The problem it solves

Publishing in two languages is slow and messy

Most small newsrooms handle bilingual publishing the hard way — copying text into Google Translate, sending drafts back and forth over email, and manually tracking which version is approved. It's error-prone and eats up hours.

🐌

Translation is a bottleneck

Writers finish an article, then wait for someone else to translate it before it can be published.

📧

No clear approval process

Reviews happen over email or chat, with no record of who approved what or when.

❌

Translation errors slip through

Machine translation isn't always accurate — without a review step, mistakes go straight to readers.

🔀

Versions get mixed up

Keeping track of which English draft matches which Filipino version is confusing across email threads.

How Balita CMS solves it

Translation built into the workflow, not added after

Instead of treating translation as a separate step, Balita CMS does it automatically the moment a writer finishes their article. The reviewer then sees both versions side by side, can fix any translation errors, and publishes with one click.

The result: what used to take hours of back-and-forth now happens inside one tool, with a clear record of every submission, review, and approval.

What comes next

This is a working prototype — here's the roadmap

This version was built in a few days to demonstrate the concept. It works end-to-end, but a production version for a real newsroom would add:

👥

Multiple real user accounts

Right now there are two demo logins. A real version would let the team create individual accounts with proper passwords.

🔔

Email notifications

Reviewers would get an email when a new article is submitted. Writers would be notified when their article is approved or rejected.

🌐

Better translation accuracy

The current translation uses free services. A production version would use a professional translation API for more natural-sounding Filipino.

📱

Mobile-friendly writing

The tool already works on mobile, but a dedicated app or improved mobile layout would make writing on the go easier.
